# 卡通农场货仓助手

面向《卡通农场》中国大陆版本的网页库存规划工具，用于记录货仓与粮仓库存、设置目标库存、展开生产链需求，并安排设备和基础原料的生产顺序。

## 功能

- 库存记录：维护当前数量、目标数量和隐藏状态。
- 缺货优先级：综合目标差额、下游需求和生产成本辅助排序。
- 生产计划：递归展开产品需求，并先扣除库存中可供下游使用的余量。
- 生产完成：增加成品库存，同时按配方扣除原料；原料不足时不会误记完成。
- 生产链查看：展示产品、原料、设备、生产时间和上下游用途。
- 数据编辑：可修改名称、设备、原材料、生产时间、目标和图标等字段。
- 自定义排序：可拖动生产设备及设备内产品，并保留个人排列习惯。
- 数据保存：浏览器自动保存使用状态，内置服务将可持久化数据写入 `data.json`。

## 运行

```bash
npm install
npm start
```

然后打开：

```text
http://localhost:8766
```

也可以直接打开 `index.html` 预览页面。直接打开和 `localhost` 使用不同的浏览器存储空间；长期使用和保存数据时，请统一通过 `npm start` 启动服务。

## 校验

```bash
npm run test:data
npm run test:core
npm run test:server
npm run test:ui
```

第一次运行 UI 测试时，可能需要安装 Playwright 浏览器：

```bash
npx playwright install chromium
```

## 数据状态

- 内置基础数据包含 461 个物品和 54 个生产分组；应用保存的修改后，当前实际生效 456 个物品。
- 物品、设备和配方以中国大陆版本的中文名称维护。
- 部分水果、鱼肉、礼券等非设备生产物会以 `0s` 或基础资源形式存在。
- `wiki_products.json` 仅作为交叉校验参考，不会自动覆盖用户保存的数据。
- 游戏版本、活动内容和中文译名可能变化，关键数据仍建议结合游戏内信息复核；发现差异时可直接在网页中修改并保存。

## 目录

- `index.html`：主应用。
- `planner-core.js`：库存需求与配方关系的纯计算模块。
- `data.json`：当前库存与生产数据。
- `icons/`：物品和设备图标。
- `server.js`：静态服务和保存接口。
- `validate.py`：数据一致性校验脚本。
- `wiki_products.json`：生产数据参考文件。
- `tests/`：浏览器冒烟测试。
