# 卡通农场货仓助手

这是一个面向《卡通农场》中国大陆版本的网页库存规划工具。当前项目以 Windows 版本 `v2.7.4` 为基础迁移整理，继续保留单页网页形态，重点用于货仓/粮仓库存记录、目标库存、缺货优先级、生产链需求和本地数据保存。

## 版本更新

### v2.7.5 - Mac 迁移与开发环境整理

- 将 Windows 桌面项目迁移为可在 Mac / Codex 环境继续开发的项目结构。
- 保留 `index.html` 和 `data.json` 的核心业务版本；它们与 GitHub 原 `v2.7.4` 主应用内容一致。
- 补齐 `icons/` 图标资源，解决远端仓库只包含 HTML 但缺少本地图标资源的问题。
- 将 `server.js` 中的 Windows 绝对路径改为 `__dirname`，避免依赖 `C:/Users/...` 路径。
- 增加 `tools/run_node.sh`，当系统没有 `node` 命令时，可回退到 Codex 自带 Node 运行环境。
- 增加 `validate.py` 和 `wiki_products.json`，用于生产数据、设备、原料和时间的校验。
- 增加 Playwright UI 冒烟测试 `tests/ui_tests.js`。
- 增加 `pnpm-lock.yaml`，记录当前迁移后的依赖状态。

## 运行方式

```bash
pnpm install
pnpm start
```

然后打开：

```text
http://localhost:8766
```

如果系统没有安装 Node.js，`pnpm start` 会通过 `tools/run_node.sh` 尝试使用 Codex 环境中的 Node。普通用户在非 Codex 环境运行时，建议安装 Node.js 18 或更高版本。

## 校验方式

```bash
pnpm test:data
pnpm test:ui
```

如果是第一次运行 UI 测试，可能需要安装 Playwright 浏览器内核：

```bash
pnpm exec playwright install chromium
```

## 目录说明

- `index.html`：主应用，包含库存、目标、优先级、生产链和编辑弹窗。
- `data.json`：当前保存数据。
- `icons/`：物品和设备图标。
- `server.js`：本地静态服务和 `/api/save` 数据保存接口。
- `validate.py`：数据一致性校验脚本。
- `wiki_products.json`：生产数据参考文件。
- `tests/`：UI 测试和旧版测试页面。
- `tools/`：迁移后辅助脚本。
- `backup/`：本地备份目录，默认不提交到仓库。

## 迁移与 AI 协作风险

这次更新涉及从 Windows 到 Mac 的开发平台迁移，也经历了 AI 协作环境变化，因此需要注意：

- 路径风险：原项目中曾存在 Windows 绝对路径，当前已改为相对项目目录，但后续新增脚本仍需避免写死本机路径。
- 依赖风险：本地使用 `pnpm` 完成依赖安装，同时保留 `package-lock.json` 兼容 npm；如果两种包管理器混用，锁文件可能再次出现差异。
- 浏览器差异：数字输入框、焦点选中、滚动和字体渲染在不同系统/浏览器上可能略有不同，UI 测试只覆盖主流程。
- 数据风险：产品名称、原料、时间来自既有中文数据和参考校验文件，仍可能存在中国大陆版本更新、翻译差异或历史录入误差。
- 图标风险：图标文件名依赖物品 ID；如果以后修改 ID，需要同步更新图标映射和测试。
- AI 协作风险：AI 更换或上下文压缩可能导致对历史决策的理解不完整。建议重要数据修改都通过 `pnpm test:data` 和人工复核双重确认。
